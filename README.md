# MoCKA
Deterministic Memory & Multi-Agent Governance Architecture

---

## What MoCKA Is

MoCKA is a deterministic, cryptographically verifiable architecture designed to address structural weaknesses in modern AI systems.

Modern AI agents are stateless.  
They forget development phases.  
They cannot deterministically reconstruct prior states.  
They cannot coordinate under shared governance without SaaS dependency.

MoCKA explores a reproducible alternative.

---

## Core Architecture

### INFIELD – Sovereign Persistent Memory
- Structured primary records (CSV / Excel / documents)
- 5W1H formal documentation regime
- Deterministic phase reconstruction
- Computational provenance (Python reproducibility)

### OUTFIELD – Cross-Agent Synchronization
- Time-series summary index
- API-optimized 75% fidelity protocol
- Cross-provider AI coordination without browser assistants

### Cryptographic Audit Backbone
- Deterministic SHA256 append-only chain
- Ed25519 key rotation with enforced retirement
- Multi-observer verification
- RFC3161 external timestamp anchoring

---

## What This Enables

- Deterministic rollback
- Phase reconstruction
- Documented AI-to-AI delegation
- Minimal SaaS dependency
- Verifiable authority succession

---

## MoCKAとは何か

MoCKA は、現代の生成AIが抱える構造的弱点を解決するために設計された、決定論的かつ暗号学的に検証可能なアーキテクチャである。

現代のAIエージェントはステートレスであり、
開発フェーズを忘れ、
過去の状態へ決定論的に戻ることができず、
SaaS依存なしで制度的協業を行うことができない。

MoCKA は、その代替となる再現可能な構造を探究する。

---

## 三層アーキテクチャ

### INFIELD – 主権的長期記憶
- CSV / Excel / 文書による原本管理
- 5W1H に基づく公式記録制度
- 決定論的フェーズ再構築
- Python による再現可能な計算履歴

### OUTFIELD – AI間同期層
- 時系列サマリーインデックス
- API制限を考慮した75%要約プロトコル
- ブラウザ依存しないAI間協調

### 暗号監査基盤
- SHA256 追記専用決定論的連鎖
- Ed25519 鍵ローテーション（退位強制）
- マルチ観測者検証
- RFC3161 外部時刻固定

---

## 実現すること

- フェーズの決定論的ロールバック
- 開発履歴の再構築
- AI間の公式委任記録
