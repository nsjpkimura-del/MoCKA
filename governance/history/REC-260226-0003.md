# Context
anchor_record.jsonはBOM許容で読めるがcalc対象外とし、代替封印anchor_record.sha256を導入した。

# Options
- Option1: anchor_record.jsonをcalc対象に含める\n- Option2: calc対象外+別ハッシュで封印 (adopted)

# Decision
anchor_record.jsonはcalc対象外。anchor_record.sha256で改変検知を成立させる。

# Rationale
自己参照・エンコーディング差分の揺らぎを排しつつ、改変検知だけは制度的に保持するため。

# Impact
- calc_summary_hashの対象定義が明確化\n- anchor改変はsha256で検知可能

# Implementation
- anchor_record.sha256を生成・保存\n- 検証手順にsha256チェックを追加
